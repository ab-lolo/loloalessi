🤖The Microcontroller Board

Arduino is an open-source electronics platform based on a microcontroller chip. Think of it as a small, programmable computer — it can read inputs from sensors (temperature, light, motion) and turn them into outputs (LEDs, motors, alarms).

The Arduino Uno (the board we use) runs on 5V, has 14 digital I/O pins, 6 analog input pins, and can be programmed via USB from a computer.

🔄Two Core Functions
void setup() {
  // Runs ONCE at startup
  // Set pin modes, start libraries
}

void loop() {
  // Runs FOREVER
  // Read sensors, control outputs
}
⚡Digital vs Analog
  • Digital: Only HIGH (5V) or LOW (0V) — like a light switch
  • Analog: Any value 0–5V, read as 0–1023 by the Arduino
  • PWM (~pins): Simulated analog output using rapid pulses
  • Pin 13 has a built-in LED — great for testing!
🔲Digital signals — only two states

A digital signal is either fully ON or fully OFF. There is nothing in between. Think of a regular light switch — it is either up (light on) or down (light off). It cannot be "half on".

On the Arduino, digital means HIGH (5V) or LOW (0V). You use digital pins to control LEDs, read buttons, and switch things on or off.

〰️Analog signals — infinite values in between

An analog signal can be any voltage between 0V and 5V — not just fully on or off. Think of a dimmer switch that lets you set a light to 10%, 50%, 80% brightness. Or a volume knob that goes from silent to full blast.

The Arduino reads analog voltages on pins A0–A5 and converts them to a number from 0 (= 0V, fully off) to 1023 (= 5V, fully on). Every value in between is possible — 312, 756, 901 — depending on exactly what the sensor is measuring.

Why 1023 and not 1024? Because counting starts at 0, not 1. If you count from 0 to 1023 you actually have 1024 different values in total — which is exactly 2¹⁰ (2 to the power of 10). The Arduino uses 10 bits internally to store the measurement, giving 1024 possible steps. Think of it like floors in a building that starts at floor 0: a building with 1024 floors goes from floor 0 to floor 1023.

〰️PWM — faking analog output

Arduino's digital pins can only output 5V or 0V — they cannot output 2.5V directly. But some pins (marked with ~) can fake an in-between voltage using a trick called Pulse Width Modulation (PWM).

PWM switches the pin ON and OFF extremely rapidly — so fast your eye cannot see it flickering. If the pin is ON half the time, the LED appears half as bright. If it is ON 80% of the time, it appears 80% bright. It is the same trick your phone uses to dim its screen.

1️⃣setup() — runs exactly once

setup() is the first function Arduino calls when it powers on or resets. It runs exactly one time, then stops. You use it to prepare everything before the main program starts.

What goes in setup():

  • Tell each pin whether it is an input or an output: pinMode(13, OUTPUT);
  • Start the Serial Monitor connection: Serial.begin(9600);
  • Initialise sensor libraries (e.g. dht.begin();)
  • Set starting values for variables if needed
void setup() {
  pinMode(13, OUTPUT);   // pin 13 will control an LED
  pinMode(7,  INPUT);    // pin 7 will read a button
  Serial.begin(9600);    // open Serial Monitor at 9600 baud
}
💡If you reset the Arduino (press the red button) setup() runs again from the beginning — but your sketch code is NOT erased.
🔁loop() — runs forever

After setup() finishes, Arduino immediately calls loop() — and when loop() reaches its closing }, it starts again from the top. This repeats thousands of times per second until you cut the power.

What goes in loop():

  • Read sensor values: analogRead(A0), digitalRead(7)
  • Make decisions: if / else based on sensor data
  • Control outputs: turn LEDs on/off, write to Serial Monitor
  • Use delay(ms) to slow things down when needed
void loop() {
  digitalWrite(13, HIGH);  // turn LED on
  delay(1000);              // wait 1 second
  digitalWrite(13, LOW);   // turn LED off
  delay(1000);              // wait 1 second
  // ← Arduino jumps back to the top and repeats
}
⚠️delay() completely pauses the Arduino — it cannot read sensors or respond to anything while waiting. For the EcoStation project we will later learn a smarter timing method using millis().
📋The complete sketch structure

Every Arduino program (called a sketch) follows this layout. The order matters — you cannot put setup() after loop().

// 1. Global variables — visible everywhere in the sketch
int ledPin = 13;
int counter = 0;

// 2. setup() — runs once
void setup() {
  pinMode(ledPin, OUTPUT);
  Serial.begin(9600);
}

// 3. loop() — repeats forever
void loop() {
  digitalWrite(ledPin, HIGH);
  delay(500);
  digitalWrite(ledPin, LOW);
  delay(500);
  counter++;
  Serial.println(counter);
}
💡Comments start with // and are completely ignored by Arduino — they exist only for humans reading the code. Use them freely to explain what each line does.

Exercise 1 — Blink the LED

Your first Arduino program. This might look simple, but controlling exactly when something turns on and off is the foundation of almost all electronic engineering.

📍 Where is pin 13? Look at the right edge of your board — a row of holes labelled 0 to 13. Pin 13 is the last one. Right next to it is a tiny LED marked L. That is the built-in LED connected directly to pin 13. No wires needed.
🌍Why this matters in real life

Controlling a pin with precise timing is exactly how real systems work. A traffic light controller does this thousands of times a day. A hospital monitor blinks an LED to show a heartbeat. A satellite blinks its status indicator in Morse code so engineers on the ground know it is alive. You are learning the same principle right now.

In physics terms: you are controlling the flow of electric current through a circuit at a precise frequency. The delay() function is your first encounter with timing — one of the most important concepts in electronics, physics, and computer science.

📖How it works

digitalWrite(pin, HIGH) sends 5V to the pin (LED ON). digitalWrite(pin, LOW) sets it to 0V (LED OFF). delay(ms) pauses for the given number of milliseconds — 1000ms = 1 second.

void setup() {
  pinMode(13, OUTPUT);
}
void loop() {
  digitalWrite(13, HIGH); // LED ON
  delay(1000);
  digitalWrite(13, LOW);  // LED OFF
  delay(1000);
}

The Serial Monitor 📟

The Serial Monitor shows text sent by your Arduino in real time. It is the most useful debugging tool you have.

📟How to use it

Add Serial.begin(9600) to setup(). Then use Serial.println() anywhere in your sketch to send text to the computer. Open it with Tools → Serial Monitor and set the baud rate to 9600.

What is baud rate? It is simply the communication speed between the Arduino and your computer — 9600 baud means 9,600 bits of data per second. The number itself is not important; what matters is that both sides use the same value. If they do not match, the Serial Monitor shows random garbage instead of your text.

⚠️Always close the Serial Monitor before uploading — it blocks the upload port and causes an error.
void setup() {
  Serial.begin(9600);
}
void loop() {
  Serial.print("Value: ");
  Serial.println(42);
  delay(500);
}
  • Serial.print() — no newline at end
  • Serial.println() — adds a newline
  • Combine: print the label, println the value
  • Set baud rate to 9600 in both places

Exercise 5 — Potentiometer & PWM

A potentiometer is a variable resistor. Turn the knob and the voltage on its middle pin changes from 0V to 5V. We read it with analogRead() and use it to control LED brightness with analogWrite().

🔩What is a Potentiometer?

A potentiometer has three legs: the two outer legs connect to 5V and GND. The middle leg (wiper) outputs a voltage between 0V and 5V depending on the knob position. analogRead(A0) reads this as a number from 0 to 1023.

We use map() to convert 0–1023 into 0–255 for analogWrite(), which sets LED brightness on a PWM pin.

Why 0–255? Same reason as 0–1023 — counting starts at 0. The Arduino uses 8 bits to store the PWM value, and 8 bits can hold exactly 2⁸ = 256 different values: 0, 1, 2 … all the way to 255. So 255 is "full on" and 0 is "off". You will see this 0–255 range often in electronics and computing — it is one of the most fundamental numbers in digital systems.

int potValue  = analogRead(A0);
int brightness = map(potValue, 0, 1023, 0, 255);
analogWrite(9, brightness);

Making Decisions with if

Until now your sketches did the same thing forever, repeating in a fixed loop. Real systems need to react to what is happening — turn the alarm on if the temperature is too high, stop the motor if the door opens. That is exactly what if does.

🌍Why decisions matter

Every intelligent system — from a thermostat to a self-driving car — is built on the same idea: measure something, compare it to a threshold, take action if the condition is met. A smoke detector does this thousands of times per second. A medical ventilator adjusts airflow based on patient breathing. The heating system in this building turns on when the temperature drops below a set point.

In physics: you are implementing a feedback loop — the foundation of control systems, which appear everywhere from simple home heating to rocket stabilisation. Programming microcontrollers gives you the tools to build these systems yourself.

📖The if statement — structure

An if statement checks whether a condition is true. If it is, the code inside the curly braces runs. If it is not, Arduino skips straight past it.

if (condition) {
  // this runs only if condition is true
}

You can also add an else branch for when the condition is false:

if (temperature > 25) {
  digitalWrite(13, HIGH);  // LED on — too warm
} else {
  digitalWrite(13, LOW);   // LED off — temperature OK
}

else if and Multiple Conditions

Two outcomes are often not enough. Real systems need to handle several different cases — and sometimes two conditions must both be true at the same time.

📖else if — chaining conditions

You can chain as many conditions as you need. Arduino checks each one in order, top to bottom, and runs the first block whose condition is true:

if (temperature > 35) {
  Serial.println("Danger — too hot!");
} else if (temperature > 25) {
  Serial.println("Warm — ventilate");
} else if (temperature > 18) {
  Serial.println("Comfortable");
} else {
  Serial.println("Too cold");
}

Exercise 1 — First Temperature Reading

Wire up the sensor and write a sketch that prints the current temperature to the Serial Monitor every second. This confirms everything is working before you add any logic.

🔌What is 1-Wire — and why does it matter for our circuit?

The DS18B20 uses a communication system called 1-Wire, invented by Dallas Semiconductor. In most protocols, a sensor needs at least two data wires (clock + data). 1-Wire uses a single wire for everything — sending commands, receiving data, even timing — with GND and VDD completing the circuit.

This is why your wiring only has three legs (GND, DQ, VDD) and one of them is data. The pull-up resistor on DQ is required because 1-Wire holds the line HIGH when idle, and the bus master (Arduino) pulls it LOW briefly to signal the start of a transmission. Without the resistor, the line floats and communication fails — which is exactly why you get -127°C without it.

💡In a real installation, you could connect 10 or 20 DS18B20 sensors all to the same single wire on pin 4 — each has a unique 64-bit ID so the Arduino can address them individually. This is what makes 1-Wire so powerful in environmental monitoring systems.
📖How the sketch works — line by line

The DS18B20 does not measure temperature continuously in the background. You have to ask it to measure, wait for the result, and then fetch it. Here is the complete sketch with comments explaining every decision:

// Import the libraries we installed in Step 3
#include <OneWire.h>
#include <DallasTemperature.h>

// Create a 1-Wire bus object on pin 4
OneWire oneWire(4);

// Connect the temperature library to that bus
DallasTemperature sensors(&oneWire);

void setup() {
  Serial.begin(9600);  // open Serial Monitor at 9600 baud
  sensors.begin();      // initialise the sensor on the bus
}

void loop() {
  // Step 1 — tell the sensor to take a measurement
  // The sensor doesn't measure "magically" in the background.
  // We have to actively ask it. This takes up to 750ms internally.
  sensors.requestTemperatures();

  // Step 2 — fetch the result from the first sensor (index 0)
  // If you had multiple sensors on the same wire, index 1, 2… would address them.
  // float means the value can have decimal places (e.g. 21.50)
  float tempC = sensors.getTempCByIndex(0);

  // Step 3 — send the data to the Serial Monitor so we can see it
  // Serial.print() outputs without a newline; Serial.println() adds one
  Serial.print("Temperature: ");
  Serial.print(tempC);
  Serial.println(" C");

  delay(1000);  // wait 1 second before the next reading
}
🛡️Professional add-on — detecting a missing sensor

When the sensor is not connected, getTempCByIndex(0) returns the special value DEVICE_DISCONNECTED_C (which is -127). Instead of silently printing a wrong number, a well-written program checks for this and reports it clearly. This is how professional firmware is written:

float tempC = sensors.getTempCByIndex(0);

// Check if the sensor responded — DEVICE_DISCONNECTED_C is -127
if (tempC == DEVICE_DISCONNECTED_C) {
  Serial.println("Sensor not found! Check wiring.");
  delay(1000);
  return;  // skip the rest of loop() and try again
}

// Only reaches here if the sensor is properly connected
Serial.print("Temperature: ");
Serial.print(tempC);
Serial.println(" C");
💡return inside loop() does not stop the program — it just exits the current run of loop() early and immediately starts the next one. This is a clean way to bail out when something is wrong.

Making Decisions with if

Until now your sketches did the same thing forever, repeating in a fixed loop. Real systems need to react to what is happening — turn the alarm on if the temperature is too high, stop the motor if the door opens. That is exactly what if does.

🌍Why decisions matter

Every intelligent system — from a thermostat to a self-driving car — is built on the same idea: measure something, compare it to a threshold, take action if the condition is met. A smoke detector does this thousands of times per second. A medical ventilator adjusts airflow based on patient breathing. The heating system in this building turns on when the temperature drops below a set point.

In physics: you are implementing a feedback loop — the foundation of control systems, which appear everywhere from simple home heating to rocket stabilisation. Programming microcontrollers gives you the tools to build these systems yourself.

📖The if statement — structure

An if statement checks whether a condition is true. If it is, the code inside the curly braces runs. If it is not, Arduino skips straight past it.

if (condition) {
  // this runs only if condition is true
}

You can also add an else branch for when the condition is false:

if (temperature > 25) {
  digitalWrite(13, HIGH);  // LED on — too warm
} else {
  digitalWrite(13, LOW);   // LED off — temperature OK
}

Your body cools itself by sweating. Sweat evaporates from your skin and carries heat away. When the air is very humid, sweat cannot evaporate efficiently — the air is already nearly saturated with water vapour. The result: you feel much hotter than the thermometer says.

The Heat Index (also called "feels like" temperature) combines dry-bulb temperature and relative humidity to estimate what the temperature actually feels like to the human body. It was developed by meteorologist Robert Steadman in 1979 and is used globally by weather services.

The MQ-2 Gas Sensor

The third and final sensor in your EcoStation: the MQ-2. It does not measure a single gas — it detects a mixture of gases and gives you a general air-quality signal. Understanding what it actually measures is crucial.

🔬What the MQ-2 actually detects

The MQ-2 is a semiconductor gas sensor. Inside it, a small coil heats a tin-oxide (SnO₂) crystal. In clean air the crystal has high electrical resistance. When certain gas molecules adsorb onto the surface, the resistance drops. The Arduino reads this change as a voltage on an analog pin.

The MQ-2 responds to several gases simultaneously:

GasSourceWhy it matters
LPG / propaneGas leaks, lightersFire and explosion risk
Methane (CH₄)Natural gas, decompositionCombustible, greenhouse gas
Smoke particlesFire, candles, cigarettesRespiratory harm, fire indicator
Carbon monoxide (CO)Incomplete combustionToxic, odourless danger
Hydrogen (H₂)Batteries, electrolysisHighly flammable
⚠️The MQ-2 does NOT directly measure CO₂ (carbon dioxide, the gas produced by human breathing). For true CO₂ measurement you would need a different sensor type (NDIR optical). The MQ-2 is a general air-quality indicator — high readings suggest air quality has degraded, but the exact gas is unknown without more analysis.
📐How the reading works

The sensor has an analog output (AO pin) connected to A0 on the Arduino. analogRead(A0) returns a value from 0 to 1023. In clean air the value is typically low (100–200). As air quality degrades the value rises — values above 400–500 suggest elevated levels of detectable gases.

⚠️The MQ-2 needs a 2-minute warm-up time before readings stabilise. Always power it up and wait before taking measurements. The heating coil consumes about 800mW — it will feel warm to the touch, which is normal. Brand-new sensor? The datasheet recommends a 24-hour burn-in on first use for fully stable readings — on day one you may see higher-than-usual values until the sensor settles in.
🌍Why use it in the classroom?

Even without identifying specific gases, the MQ-2 provides useful real-world data. A classroom with poor ventilation accumulates body odour, trace gases from cleaning products, and particles from whiteboard markers. A rising MQ-2 reading over the course of a lesson is a measurable proxy for ventilation quality — exactly what the Erasmus+ EcoStation is designed to reveal.

Professional air quality monitors use the same principle, just with more sophisticated and more specific sensors. The data you collect in this classroom is the same type of data used in environmental health research.

// ── EcoStation: Classroom Environmental Monitor ──────────────────
// Combines DS18B20 (temperature), DHT22 (temp + humidity),
// and MQ-2 (air quality) into one complete monitoring system.

#include <OneWire.h>
#include <DallasTemperature.h>
#include "DHT.h"

// ── Sensor objects ────────────────────────────────────────────────
OneWire oneWire(4);                    // 1-Wire bus on pin 4
DallasTemperature ds18(&oneWire);      // DS18B20 temp sensor
DHT dht(3, DHT22);                    // DHT22 on pin 3

void setup() {
  Serial.begin(9600);
  ds18.begin();   // initialise DS18B20
  dht.begin();    // initialise DHT22
  pinMode(13, OUTPUT);  // alarm LED
  Serial.println("EcoStation ready — warming up sensors...");
}

void loop() {
  // ── 1. Read DS18B20 temperature ──────────────────────────────
  ds18.requestTemperatures();  // ask sensor to measure
  float tempDS = ds18.getTempCByIndex(0);

  // ── 2. Read DHT22 temperature + humidity ─────────────────────
  float h    = dht.readHumidity();
  float tDHT = dht.readTemperature();

  // ── 3. Read MQ-2 air quality (0 = clean, 1023 = very high) ──
  int gas = analogRead(A0);

  // ── 4. Guard: check for bad DHT22 reading ────────────────────
  if (isnan(h) || isnan(tDHT)) {
    Serial.println("DHT22 error — retrying...");
    delay(2000);
    return;  // skip this iteration
  }

  // ── 5. Calculate heat index ───────────────────────────────────
  float hi = dht.computeHeatIndex(tDHT, h, false); // Celsius

  // ── 6. Print all readings ─────────────────────────────────────
  Serial.println("────────────────────────");
  Serial.print("DS18B20 temp:  "); Serial.print(tempDS); Serial.println(" C");
  Serial.print("DHT22 temp:    "); Serial.print(tDHT);   Serial.println(" C");
  Serial.print("Humidity:      "); Serial.print(h);      Serial.println(" %");
  Serial.print("Feels like:    "); Serial.print(hi);     Serial.println(" C");
  Serial.print("Air quality:   "); Serial.println(gas);

  // ── 7. Alarm: LED on if gas level elevated ────────────────────
  if (gas > 500) {
    digitalWrite(13, HIGH); // air quality alert
    Serial.println("  ⚠ GAS ALERT");
  } else {
    digitalWrite(13, LOW);  // all clear
  }

  delay(2000);  // DHT22 needs at least 2s between readings
}
